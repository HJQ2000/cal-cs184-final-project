Brief summarize

We studied the codebase for project 3 and tried to add a texture model into the codebase. We find this difficult to start with, so we learn to write a simplified version of ray tracer process with far less options. We collected learning materials to do texture mapping for regular shape objects such as sphere and cube. By now, we just have spheres in different colors.

Goals and Deliverables

What we have for now.

We are running back at of initial plan, and find texture mapping to irregular shape hard to understand. We change our goal to first have image of solar system with sphere of different size and texture. If we still have time, we will try to deal with irregular shape.

Our logic by now is to first load the image of texture, then convert the image into uv texture coordinate, then map the uv coordinate to 3D sphere xyz space, and finally output the resulting image to file. To load the image, we import stbi_load class.
